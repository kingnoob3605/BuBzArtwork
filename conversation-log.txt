SESSION 3 — SUPABASE BACKEND PLAN (NOT YET IMPLEMENTED)
--------------------------------------------------------------------------------

User plans to use Supabase as backend. No separate API server needed.

SUPABASE STORAGE BUCKETS:
  - "gallery"     — admin uploads artwork images
  - "submissions" — anonymous canvas drawing uploads (PNG blobs)

SUPABASE DATABASE TABLES:
  - artworks    (id, title, description, date, tags, nsfw, image_url)
  - submissions (id, type TEXT 'drawing'|'message', data TEXT, created_at)
  - comments    (id, artwork_id, text, created_at)

SUPABASE CLIENT (no backend server needed):
  <script src="https://cdn.jsdelivr.net/npm/@supabase/supabase-js@2"></script>
  const supabase = window.supabase.createClient('YOUR_URL', 'YOUR_ANON_KEY')

ROW LEVEL SECURITY (RLS) to configure:
  - artworks:    public READ, authenticated-only INSERT/UPDATE/DELETE
  - submissions: public INSERT, authenticated-only READ/DELETE
  - comments:    public READ + INSERT
  - submissions bucket: public upload, authenticated-only read/delete

MIGRATION (localStorage -> Supabase):
  Currently                        | With Supabase
  artworks array in data.js        | SELECT * FROM artworks
  Admin adds art -> localStorage   | INSERT INTO artworks + Storage upload
  Canvas drawing -> base64 local   | Upload PNG blob -> save URL in submissions
  Message -> localStorage          | INSERT INTO submissions
  Comments -> localStorage         | INSERT INTO comments with artwork_id

Optional API layer only needed for:
  - Email notifications on new submissions
  - Rate limiting
  - Image moderation

SESSION 8 — AGE VERIFICATION IMPROVEMENTS
--------------------------------------------------------------------------------

User was worried the age gate wasn't strong enough.

WHAT WAS IMPROVED:

1. 30-day expiry on confirmation
   BEFORE: localStorage.getItem('age-confirmed') — lasted forever
   AFTER:  Stores timestamp in 'age-confirmed-ts', checks if > 30 days old.
           If expired, shows gate again.

2. "Just this visit" option
   - Small underlined link below the main buttons
   - Uses sessionStorage instead of localStorage
   - Gate re-appears every new browser session
   - Button text: "Just this visit (don't remember me)"

3. NSFW cards blurred by default in gallery
   - All NSFW art cards show blurred (filter: blur(18px) brightness(0.6))
   - Overlay shows 🔞 NSFW + "Hover to preview"
   - Hovering the card smoothly unblurs the image (CSS transition)
   - Clicking still opens lightbox as normal
   - This is the industry standard (Twitter, DeviantArt use same pattern)

CHANGES MADE:

index.html — age gate buttons:
  - "Enter" button now calls confirmAge(false)
  - "Leave" button label changed to "I'm under 18 — Leave"
  - Added <button class="btn-session-only"> below buttons

app.js — init():
  - Checks both sessionStorage('age-session') and localStorage expiry
  - Shows gate only if both are missing/expired

app.js — confirmAge(sessionOnly):
  - If sessionOnly=true: sets sessionStorage 'age-session'
  - If sessionOnly=false: sets localStorage 'age-confirmed-ts' = Date.now()

app.js — renderGallery():
  - NSFW cards wrapped in .art-card-img-wrap.is-nsfw
  - Injects .nsfw-blur-overlay with icon, label, hint text

style.css — new classes:
  - .btn-session-only: underline link style, opacity 0.6
  - .art-card-img-wrap: position relative, overflow hidden
  - .art-card-img-wrap.is-nsfw img: blur(18px) by default
  - .art-card:hover .is-nsfw img: blur(0) unblur on hover
  - .nsfw-blur-overlay: centered flex, fades out on card hover
  - .nsfw-blur-icon, .nsfw-blur-label, .nsfw-blur-hint

NOTE: All age verification is client-side only. A determined person can bypass
it. True enforcement requires a backend with ID/credit card verification.
For a personal art site, this is the realistic standard.
